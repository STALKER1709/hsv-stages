Objectif de cette leçon

Comprendre et utiliser les relations Eloquent pour lier les modèles entre eux (one-to-one, one-to-many, many-to-many).

1. La relation hasMany

Dans Laravel Eloquent, hasMany définit une relation un-à-plusieurs. Par exemple, un Utilisateur peut avoir plusieurs Posts.

// app/Models/User.php
class User extends Model
{
    public function posts(): HasMany
    {
        return $this->hasMany(Post::class);
    }
}

// Utilisation
$user = User::find(1);
$posts = $user->posts; // Collection de tous ses posts

2. La relation belongsTo

L'inverse de hasMany — un Post appartient à un User.

// app/Models/Post.php
class Post extends Model
{
    public function user(): BelongsTo
    {
        return $this->belongsTo(User::class);
    }
}

// Utilisation
$post = Post::find(1);
$auteur = $post->user; // L'utilisateur qui a créé ce post
echo $auteur->name;

3. Eager Loading – Éviter le problème N+1

Problème N+1

Sans eager loading, pour 100 users, Laravel fait 101 requêtes SQL (1 pour les users + 100 pour leurs posts).

// ❌ Mauvais – problème N+1
$users = User::all();
foreach ($users as $user) {
    echo $user->posts->count(); // 1 requête par user !
}

// ✅ Bien – Eager loading avec with()
$users = User::with('posts')->get();
foreach ($users as $user) {
    echo $user->posts->count(); // 0 requête supplémentaire
}

Dans votre projet de stage, créez une relation entre le modèle Stagiaire et Evaluation. Un stagiaire peut avoir plusieurs évaluations.
